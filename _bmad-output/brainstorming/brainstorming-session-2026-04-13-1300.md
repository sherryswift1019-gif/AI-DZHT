## Session Overview

**主题：** AI-DZHT 平台 Agent 管理模块
**目标：** 功能需求 + 用户体验为主，兼顾风险/边界探索
**产出：** 25 个功能需求想法 + 15 个 UX 想法 + 关键架构决策 + 风险清单

### 核心范围共识

Agent 管理模块 = **Agent 模板注册表 + 能力编排系统**，不是运维监控系统。

```
BMAD 命令库（BP/DR/CA/DS...可扩展）
        ↓ 勾选能力 + AI辅助生成提示词
    Agent 模板（内置8个 + 自定义）
        ↓ 实例化时注入三层上下文
    Agent 实例（每个需求 = 独立会话）
        ↓ 由指挥官动态调度顺序
    需求工作流执行
```

## 主题二：Agent 模板创建与编辑

**核心洞察：** 创建 Agent = 勾选能力 + AI 辅助生成提示词 + 版本管理，三者缺一不可

| # | 功能想法 | 优先级 |
|---|---|---|
| F04 | **三步创建向导**：① 基本信息（名称/颜色/图标）→ ② 能力勾选 → ③ 提示词（AI生成+调整），步骤间可前后跳转 | P0 |
| F05 | **AI 辅助提示词生成**：选完能力后一键"AI生成提示词"，根据勾选命令自动生成结构化草稿供修改 | P0 |
| F06 | **提示词分段结构化编辑器**：四块独立编辑——① 身份定义 ② 能力边界 ③ 行为规范 ④ 上下文规则，各块可单独 AI 优化 | P1 |
| F07 | **提示词质量评分**：保存前 AI 自动评分（清晰度/能力覆盖/边界约束/命令匹配度）并给出改进建议 | P2 |
| F08 | **Agent 卡片实时预览**：创建过程中右侧实时显示 Agent 卡片预览，随填写内容同步更新 | P1 |
| F09 | **Agent 版本管理**：每次修改自动生成新版本，保留历史可回滚；工作流引用可绑定特定版本 | P1 |

## 主题四：能力重复发现与防冗余

**核心洞察：** "Agent 太多、能力重复、不知道哪个 Agent 有我要的能力" 是核心用户痛点

| # | 功能想法 | 优先级 |
|---|---|---|
| F12 | **能力反查（命令→Agent）**：输入命令代码（如 QA），系统列出所有拥有该能力的 Agent | P0 |
| F13 | **创建时重复警告**：选完能力后系统自动比对，若与现有 Agent 高度重叠（>80%）给出警告和 Fork 建议 | P0 |
| F14 | **能力热力图总览**：Agent 列表页提供"能力覆盖矩阵"视图——横轴命令、纵轴 Agent，色块显示覆盖与重复情况 | P2 |
| F15 | **智能能力推荐**：输入 Agent 描述意图，AI 推荐该勾选哪些命令，并提示相似现有 Agent | P1 |
| F16 | **能力独占标记**：某些命令可标记"建议单 Agent 专属"，多 Agent 重复持有时给出视觉提醒 | P2 |

## 主题五：Agent 生命周期管理（停用/删除/共享）

**核心决策已确认：**
- 停用：允许，停用前必须检查依赖工作流
- 删除：与停用同等检查逻辑
- 共享：加开关，三级范围（仅本团队/平台内共享/申请后使用）
- 提示词修改：有运行中实例时锁定，停止后方可修改

| # | 功能想法 | 优先级 |
|---|---|---|
| F17 | **停用前依赖检查**：点击停用触发扫描，列出所有引用该 Agent 的工作流，提供"前往修改"或"强制停用（管理员）"两个出口 | P0 |
| F18 | **提示词修改锁定**：有运行中实例时编辑按钮自动灰化，tooltip 显示占用实例数，点击查看是哪些需求在用 | P0 |
| F19 | **共享三级范围控制**：① 仅本团队 ② 平台内自由使用 ③ 平台内需申请；共享 Agent 显示"来源团队"水印 | P1 |
| F20 | **共享 Agent 版本订阅**：B 团队使用 A 团队共享 Agent 后，A 更新版本时 B 收到通知，可选择跟随或固定版本 | P2 |
| F21 | **停用不破坏历史**：已完成工作流保留 Agent 快照（名称/版本/当时提示词），停用后历史记录仍完整可查 | P0 |

## 主题六：Agent 实例化与三层上下文注入

**核心架构决策：Agent = 模板，每个需求 = 独立会话实例**

```
Agent基础提示词（模板层）
    + 项目上下文（行业/技术栈/规范）
    + 需求上下文（PRD摘要/目标/关联Story）
    = Agent 实例（完全隔离的独立会话）
```

| # | 功能想法 | 优先级 |
|---|---|---|
| F22 | **项目级上下文配置**：每个项目可配置"通用上下文"（行业/技术栈/团队规范），自动附加到该项目所有 Agent 实例提示词前缀 | P0 |
| F23 | **上下文注入预览**：Agent 详情页提供"实例化预览"——选择项目+需求，展示该 Agent 实际收到的完整提示词，用于调试优化 | P1 |
| F24 | **会话隔离 + 经验萃取**：实例间完全隔离，需求完成后可主动提炼关键决策为"团队知识片段"供后续实例选择性引用 | P2 |
| F25 | **Agent 实例完整会话回放**：每个需求下每个 Agent 的完整会话历史、思考链路、命令序列可查看和回放 | P1 |

## 主题七：Agent 管理界面 UX

**核心 UX 原则：** Agent 管理是"后台配置"场景，优先清晰和可维护性，而非炫酷

| # | UX 想法 | 优先级 |
|---|---|---|
| UX01 | **Agent 列表页双视图**：默认卡片视图（含能力标签/状态徽章）+ 可切换能力矩阵视图 | P1 |
| UX02 | **能力搜索优先入口**：搜索栏支持命令代码搜索（输入`QA`过滤所有有QA能力的 Agent），结果高亮匹配标签 | P0 |
| UX03 | **"我要这个能力"引导流**：列表页入口——选命令→列出匹配 Agent→三选一（直接用/Fork/新建） | P0 |
| UX04 | **Agent 状态徽章四维**：来源（内置/自定义/共享来源）+ 运行状态（运行中N个/空闲/已停用）+ 锁定状态 + 共享范围 | P0 |
| UX05 | **Agent 详情页双标签**：「模板配置」（能力/提示词/版本历史）+ 「运行实例」（当前多少实例/在哪些项目/各自状态） | P1 |
| UX06 | **相似度对比卡**：创建保存前若检测到相似 Agent，弹出左右对比卡，差异高亮，用户最终决策 | P1 |
| UX07 | **创建时能力分组面板**：命令按4阶段分4列，勾选时右侧实时更新职责摘要预览 | P0 |

## 风险与边界决策汇总

| 风险场景 | 已确认的处理策略 |
|---|---|
| 停用/删除 Agent 但有工作流在用 | 先检查依赖，列出影响工作流，不允许直接删；提示修改工作流后再操作 |
| 修改提示词时有实例正在运行 | 自动锁定，停止所有运行中实例后方可修改 |
| Agent 能力重复冗余 | 创建时重复警告 + 能力反查 + 热力图 |
| 内置 Agent 误操作 | 内置标记保护，不可删除，只能 Fork |
| 共享 Agent 被随意修改 | 共享方控制是否允许 Fork，版本订阅可不跟随 |
| 提示词质量低 | AI 辅助生成 + 质量评分 + 结构化编辑 |
