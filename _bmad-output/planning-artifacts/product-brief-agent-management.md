# 功能简介：AI-DZHT Agent 管理模块

## 模块定位

Agent 管理模块是 AI-DZHT 平台的**能力注册中心**与 **Agent 编排基础设施**。它不是运维监控面板，而是 Agent 的"人事部门"——负责 Agent 的定义、能力编排、生命周期管理与上下文注入策略，为平台上运行的所有研发工作流提供可靠的 Agent 供给。

---

## 问题

AI-DZHT 平台支持多项目、多需求并行运行，每个需求的实现都依赖 8 个专职 Agent 的协作。随着平台规模扩大，一个系统性的问题开始浮现：

**Agent 管理缺乏统一治理。** 平台内置 8 个标准 Agent，但不同项目有不同的规范诉求——金融类项目需要"合规优先"的产品 Agent，游戏类项目需要"速度导向"的产品 Agent。没有能力编排机制，每次只能修改全局配置，改了这个项目就影响了那个项目。

**能力发现成本极高。** 当管理员想创建一个新 Agent 时，他无法快速知道"现有 Agent 里有没有已经具备我想要能力的"。能力散落在各 Agent 配置里，只能逐个查看，造成大量重复冗余 Agent 堆积。

**提示词质量无保障。** Agent 的行为质量完全取决于提示词写得好不好。当前没有结构规范、没有质量检查、也没有版本追踪——一次随意的修改可能让正在运行的工作流行为突变，且无法回滚。

**上下文注入缺失导致 Agent 行为通用化。** 每个需求都有其行业背景、技术规范和业务约束，但 Agent 被以同一配置运行在所有需求上，无法感知上下文差异，产出必然是"通用的"而非"契合的"。

---

## 解决方案

### 核心架构范式：模板 / 实例分离

```
BMAD 命令库（可扩展的原子能力单元）
        ↓ 勾选能力 + AI 辅助生成提示词
    Agent 模板（内置 8 个 + 用户自定义）
        ↓ 实例化时注入三层上下文
    Agent 实例（每个需求 = 独立隔离会话）
        ↓ 由指挥官 Agent 动态调度执行顺序
    需求工作流
```

**Agent 模板**是配置层，定义能力与行为规范；**Agent 实例**是运行层，每个需求独立会话完全隔离。修改模板不影响正在运行的实例，历史执行记录完整可追溯。

## 核心功能

### 1. BMAD 命令库管理
系统维护所有 BMAD 方法论命令的标准列表（BP/DR/CA/DS/QA 等），每条命令含名称、描述和所属阶段（分析/规划/架构/实现）。内置命令为系统标准，不可删除。命令库扩展（新增自定义命令）为 P1 功能，由管理员在二期添加，新命令即时生效于 Agent 能力选择。

### 2. Agent 能力编排（三步向导）
创建 Agent 分三步：
- **步骤 1**：填写基本信息（名称、颜色标识、图标）
- **步骤 2**：勾选能力（BMAD 命令按4阶段分组展示，勾选时实时预览职责摘要）
- **步骤 3**：提示词配置（AI 根据所选能力自动生成结构化草稿，人工微调）

提示词采用**四块结构化编辑**：① 身份定义 ② 能力边界 ③ 行为规范 ④ 上下文规则，各块独立维护，支持单块 AI 优化。

### 3. 能力发现与防冗余
- **能力反查**：输入命令代码（如 `QA`），立即列出所有拥有该能力的 Agent
- **创建时重复警告**：能力组合与现有 Agent 高度重叠时自动提示并推荐 Fork
- **"我要这个能力"引导流**：先找已有 → 不满足再 Fork → 都不合适再新建

### 4. 内置 Agent 保护与 Fork 机制
8 个内置 Agent 标记"系统保护"不可删除，但支持 Fork 出变体——Fork 后可在原有能力基础上增减命令、修改提示词，变体与原版建立血缘关联，支持差异对比。

### 5. 三层上下文注入
每次需求执行时，Agent 实例自动叠加三层上下文：
- **Agent 基础提示词**（模板层）
- **项目上下文**：行业领域、技术栈、团队规范（项目级配置，一次设定全项目复用）
- **需求上下文**：PRD 摘要、需求目标、关联 Story 列表（每次运行自动注入）

层级越具体，优先级越高。Agent 行为随上下文自适应，而无需为每个项目创建独立 Agent。

### 6. Agent 生命周期管理
- **停用**：停用前自动扫描依赖工作流，有关联时提示先修改工作流再操作
- **删除**：同等检查逻辑，内置 Agent 不可删除
- **共享**：三级范围控制（仅本团队 / 平台内共享 / 需申请使用），发布者保留 Fork 权限控制
- **提示词修改保护**：有运行中实例时编辑入口自动锁定，停止所有实例后方可修改
- **历史快照**：停用/删除不破坏历史记录，已完成工作流保留当时的 Agent 版本快照

### 7. 版本管理
每次修改 Agent 自动生成版本记录，历史版本可回滚；工作流模板引用 Agent 时可绑定特定版本，保障执行稳定性。

### 8. Agent 状态徽章系统
Agent 列表每张卡片用四维徽章表达完整状态：**来源**（系统内置 / 自定义 / 共享-来源团队）+ **运行状态**（运行中 N 个实例 / 空闲 / 已停用）+ **锁定状态**（提示词是否可编辑）+ **共享范围**（仅本团队 / 平台共享 / 需申请）。一卡可读，无需进入详情页即知能否操作。

### 9. Agent 详情页与实例视图（P1）
Agent 详情页包含两个标签：「**模板配置**」（能力/提示词/版本历史）和「**运行实例**」（当前被多少需求使用、所在项目/需求、各实例状态、是否触发锁定）。创建/编辑页右侧提供 Agent 卡片**实时预览**，随填写内容同步更新，所见即所得。保存前若检测到相似 Agent，弹出左右**对比卡**，差异高亮，用户最终决策。

### 10. Agent 实例会话回放（P1）
每个需求下每个 Agent 的完整执行记录可查看和回放：完整会话历史、思考链路、调用的命令序列。用于复盘产出质量问题的根因——精确定位是哪一步决策出现偏差，而非只知道"结果不对"。

## 目标用户

**平台管理员**：负责维护内置 Agent 配置、审批共享 Agent、管理 BMAD 命令库扩展。关注系统稳定性与 Agent 质量标准。

**项目/团队负责人**：为自己的项目创建定制化 Agent 变体或自定义 Agent，配置项目级上下文。关注"Agent 的行为是否符合我们项目的规范和风格"。

---

## 成功衡量

| 指标 | 目标 |
|---|---|
| Agent 重复率 | 平台内能力重叠度 > 80% 的 Agent 数量 < 5% |
| 提示词创建时长 | 新建 Agent 的提示词配置时间 < 10 分钟（含 AI 生成） |
| 实例隔离有效性 | 跨需求会话污染事件 = 0 |
| 生命周期操作安全 | 因 Agent 停用/删除导致的工作流中断事件 = 0 |

---

## 范围边界

**本模块包含：**
- BMAD 命令库的增删管理
- Agent 模板的 CRUD（含能力编排/提示词/版本）
- Agent 共享与 Fork 机制
- 三层上下文注入配置
- Agent 生命周期（停用/删除/锁定）

**本模块不包含：**
- 需求视角的 Pipeline 执行详情（Agent 输出内容、审批节点操作，属于工作流执行模块）
- 工作流的编排与调度逻辑（属于工作流配置模块）
- 需求/项目管理（属于项目管理模块）

> **说明：** Agent 管理模块提供 **Agent 视角**的实例观测——即"某个 Agent 模板当前被多少需求使用、各实例健康状态、是否处于锁定"。这与工作流执行模块的"需求视角"（某个需求的 Pipeline 进展和产出）是互补而非重复的关系。

## 优先级分期建议

**P0 — MVP（必做）：**
BMAD 命令库 · 三步创建向导 · AI 生成提示词 · 能力反查 · 内置 Agent 保护 · 停用前依赖检查 · 提示词锁定 · 历史快照 · 项目级上下文配置

**P1 — 二期（重要）：**
命令库扩展 · 结构化提示词编辑器 · Agent 版本管理 · Fork & 变体管理 · 能力智能推荐 · 共享三级范围控制 · 上下文注入预览 · Agent 实例视图（详情页双标签）· Agent 卡片实时预览 · 相似 Agent 对比卡 · 会话回放

**P2 — 三期（增强）：**
提示词质量评分 · 能力热力图总览 · 能力独占标记 · 共享版本订阅 · 会话经验萃取
