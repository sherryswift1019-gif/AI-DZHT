## 4. 实现亮点

### 4.1 互斥分类算法
`computeStatCounts` 将 `running` 状态的需求按当前 pipeline 步骤二次分类为 `pendingApproval`、`pendingInput`、`running`，消除原方案中"执行中"与"待审批"的维度重叠。每条需求 **有且仅有一个** 归属桶，求和恒等于总数。

### 4.2 双分组视觉层次
- **介入组**（暖色 `--attention-surface` 背景）：待审批 / 阻塞 / 待输入——决策者一眼看到需要行动的条目
- **进度组**（中性区）：执行中 / 待执行 / 已完成——常规流转
- **总数标签**：锚定参照

### 4.3 设计 Token 双主题适配
`--amber` 和 `--attention-surface` 同时在 light/dark 主题定义，6% 透明度背景在两种主题下均保持视觉一致性。
