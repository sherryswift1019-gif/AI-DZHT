# TRV Testability Review — Validation Checklist

## Input Coverage

- [ ] PRD loaded or user-provided requirements captured
- [ ] All functional requirements extracted
- [ ] Acceptance criteria identified for each requirement

## Assessment Quality

- [ ] Every requirement scored on all 4 dimensions
- [ ] Scores are consistent (same criteria = same score across requirements)
- [ ] Green/Yellow/Red classification correct per score
- [ ] No requirement left unscored

## Improvement Suggestions

- [ ] Every Yellow/Red requirement has a concrete rewrite suggestion
- [ ] Suggestions are actionable (can be directly applied to PRD)
- [ ] Suggestions improve the specific weak dimension
- [ ] No vague suggestions ("needs more detail")

## Overall Assessment

- [ ] Overall score calculated correctly
- [ ] Category counts sum to total requirements
- [ ] Top systemic issues identified
- [ ] Recommended actions prioritized

## Output

- [ ] Report saved to configured output path
- [ ] Report follows required section structure
- [ ] All tables properly formatted

---

**Quality Bar**: After reading this review, the PM should be able to revise all Red requirements in under 30 minutes without asking for clarification.
