# LJ LLM Quality Judge — Validation Checklist

## Collection

- [ ] Test files collected and paired with source files
- [ ] Scope appropriate (all files or user-specified subset)

## Evaluation

- [ ] Every file scored on all 5 dimensions
- [ ] Scores backed by specific code examples
- [ ] Scores are calibrated (same quality = same score across files)
- [ ] Suite averages calculated correctly

## Improvement Suggestions

- [ ] Weakest dimensions identified
- [ ] Before/after code examples provided
- [ ] Suggestions ranked by impact
- [ ] Strengths acknowledged

## Output

- [ ] Report saved to configured path
- [ ] Per-file breakdown table included
- [ ] Top improvement opportunities specific and actionable

---

**Quality Bar**: After reading this report, a developer should fix the top 3 issues and measurably improve suite quality.
