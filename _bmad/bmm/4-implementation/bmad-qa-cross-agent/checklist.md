# XA Cross-Agent Quality — Validation Checklist

## Data Gathering

- [ ] Available quality data loaded (TH/CT/QG/TS/LJ)
- [ ] Missing data sources noted

## Role-Specific Feedback

- [ ] Dev feedback focuses on code-level issues
- [ ] PM feedback focuses on requirement-level issues
- [ ] SM feedback focuses on sprint-level impact
- [ ] Architect feedback focuses on structural issues
- [ ] Each role gets 3-5 specific, actionable items
- [ ] Language appropriate for each role

## Context Updates

- [ ] Suggested lessons include scope and correctApproach
- [ ] Suggested rules are enforceable and measurable

## Output

- [ ] Report saved to configured path
- [ ] Priority actions listed across all roles

---

**Quality Bar**: Each agent role should be able to take immediate action on at least one item without asking for clarification.
